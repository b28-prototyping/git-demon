## High-Speed Perspective Overhaul

The renderer currently uses a single-point pseudo-perspective with linear depth interpolation. This produces a flat, arcade-like look that lacks the visceral sense of speed found in classic racing games (OutRun, F-Zero, Wipeout). Key deficiencies:

1. **Linear depth mapping** — `screen_y = lerp(horizon, bottom, depth_scale)` produces uniform spacing. True perspective uses `1/z`, which compresses far geometry and stretches near geometry, creating the sensation that the ground is rushing under you.
2. **No hills or pitch** — the road is an infinite flat plane. Segment-based road geometry (à la OutRun) allows hills, dips, and crests that reveal/conceal the road ahead and add drama.
3. **No parallax layering** — sky, terrain, and road all share the same projection. Distant elements (stars, clouds) should scroll slower than near elements (terrain silhouettes) to reinforce depth.
4. **Static camera** — FOV widens slightly with speed but there's no pitch, bank, or lateral lag. A dynamic camera that reacts to speed and curvature sells the feeling of a physical vehicle.
5. **Inconsistent sprite scaling** — sprites use `scale²` for width which doesn't match any consistent projection model, and road grid vs sprite lane widths are mismatched.

### Design Principles

- **Segment-based road**: Divide the road into discrete Z-segments, each with curvature (dx) and pitch (dy) values. Project segments back-to-front using cumulative offsets. This is the classic pseudo-3D technique that OutRun, Lotus, and countless racers use.
- **1/z projection**: Replace linear lerp with reciprocal-depth scaling so near geometry dominates screen space and far geometry compresses toward the horizon.
- **Unified projection function**: One canonical `project(z_world) → (screen_x, screen_y, scale)` used by road, sprites, terrain, and effects.
- **Camera as first-class entity**: A `Camera` struct with position, pitch, yaw, and FOV that all systems read from. Speed, curvature, and git events drive camera dynamics.
- **Preserve the vibe**: The neon-grid cyber aesthetic is core. Improvements should amplify the existing look, not replace it.
