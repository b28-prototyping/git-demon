## Context

`RepoSeed` is computed once at startup from the full git history. It provides the deterministic identity of the visual world: accent hue, terrain roughness, per-author colors, speed base. Same repo always produces same visual world.

## Acceptance Criteria

- [ ] `RepoSeed::compute()` opens a git repo via `git2::Repository::open` and walks full history
- [ ] `accent_hue` is deterministically derived from remote origin URL (or first commit SHA if no remote)
- [ ] Same repo always produces the same `accent_hue` value
- [ ] `terrain_roughness` is derived from commit frequency variance, clamped to `[0.1, 1.0]`
- [ ] `author_colors` maps each unique author name to a stable `Rgba<u8>` color via name hash
- [ ] `repo_name` is extracted from remote URL basename (stripping `.git`) or falls back to directory name
- [ ] `total_commits` is the full count of commits in history
- [ ] Handles edge cases: empty repo (0 commits), no remote, detached HEAD
- [ ] HSL→RGB color conversion is correct (unit tested)
