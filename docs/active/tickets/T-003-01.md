## Context

Roadside objects are rendered as depth-projected sprites. Each object has a world-space position `(lane, z_world)` that projects to screen coordinates. Sprites scale quadratically with depth for an OutRun feel. Text on sprites appears only when close enough.

## Acceptance Criteria

- [ ] `project()` converts `(z_world, lane)` to screen `(x, y, scale)` with correct perspective
- [ ] Projection accounts for road curvature (curve_offset applied to sprite x-positions)
- [ ] Sprites sort back-to-front by `z_world` descending for correct overdraw
- [ ] Sprite size scales quadratically: `base * scale²`
- [ ] `CommitBillboard`: colored rect with author color, white text overlay
- [ ] `AdditionTower`: vertical rect, height proportional to `sqrt(lines)`
- [ ] `DeletionShard`: crimson vertical rect
- [ ] `TierGate`: full-width neon arch with tier name text
- [ ] `VelocitySign`: yellow diamond with c/min number
- [ ] Text suppressed when `depth_scale < 0.35`, 1× at ≥0.35, 2× at ≥0.65
- [ ] Lane positioning: Left/Right at `road_half * 1.15`, Center at road center
- [ ] Objects beyond draw distance or behind camera are culled
