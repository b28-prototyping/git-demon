## Context

The git poller runs on a background `std::thread`, waking every `--interval` seconds to walk recent commits and compute diff stats. It sends `PollResult` over a `crossbeam_channel::Sender`. The main render loop drains this channel non-blocking.

## Acceptance Criteria

- [ ] `GitPoller::spawn()` starts a background thread that polls at the configured interval
- [ ] First poll happens immediately on spawn (not after first interval delay)
- [ ] Each poll walks commits within `--window` minutes using `git2` revwalk with `Sort::TIME`
- [ ] `PollResult` includes: commit list, commits_per_min, lines added/deleted, files changed
- [ ] Each `CommitSummary` includes: sha_short (7 chars), first line of message, author, diff stats, timestamp
- [ ] Diff stats use `git2::Diff::stats()` (tree-to-tree diff against parent)
- [ ] Thread exits cleanly when the receiver is dropped
- [ ] No subprocess calls — pure `git2` API
- [ ] Handles repos with no commits gracefully (empty PollResult)
