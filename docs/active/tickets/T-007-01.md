## Context

Every subsystem (road, sprites, terrain, sky, effects) currently computes its own ad-hoc projection with slightly different formulas. Road grid lines use `lerp(0, max_half * 1.5, depth)` while sprites use `lerp(ROAD_MIN_HALF, max_half, depth)`. Speed lines radiate from `horizon_y * 0.6` while the road vanishing point is at `horizon_y`. This ticket introduces a single `Camera` struct and a canonical projection function that all systems share.

## Implementation

### 1. Add `Camera` struct (`src/world/camera.rs`, new file)

```rust
pub struct Camera {
    pub z: f32,             // world-Z position (currently world.camera_z)
    pub pitch: f32,         // vertical tilt in radians (positive = look up)
    pub yaw_offset: f32,    // lateral offset from curvature lag
    pub fov_scale: f32,     // 1.0 = default, >1.0 = wider FOV
    pub horizon_ratio: f32, // screen fraction from top to horizon (computed)
    pub draw_distance: f32, // max visible Z depth
}
```

### 2. Add canonical `project()` function

```rust
impl Camera {
    /// Project a world-Z coordinate to screen space.
    /// Returns (screen_y, depth_scale) where depth_scale ∈ (0, 1].
    /// Uses 1/z projection: depth_scale = near_plane / z_rel.
    pub fn project(&self, z_world: f32, screen_h: u32, horizon_y: u32) -> Option<(f32, f32)> {
        let z_rel = z_world - self.z;
        if z_rel <= NEAR_PLANE || z_rel > self.draw_distance {
            return None;
        }
        let depth_scale = (NEAR_PLANE / z_rel).clamp(0.0, 1.0);
        let screen_y = horizon_y as f32 + (screen_h as f32 - horizon_y as f32) * depth_scale;
        Some((screen_y, depth_scale))
    }
}
```

Where `NEAR_PLANE` is calibrated so that `depth_scale = 1.0` at the camera's feet (bottom of screen).

### 3. Add `project_x()` for lateral positioning

```rust
    /// Project a world-X offset at a given depth_scale to screen-X.
    pub fn project_x(&self, x_offset: f32, depth_scale: f32, screen_w: u32) -> f32 {
        let cx = screen_w as f32 / 2.0 + self.yaw_offset * depth_scale;
        cx + x_offset * depth_scale
    }
```

### 4. Integrate into `WorldState`

Add `camera: Camera` field to `WorldState`. Move `camera_z`, `draw_distance`, and horizon calculation into `Camera::update(dt, speed, curve_offset)`. Remove the scattered constants.

### 5. Migrate road.rs to use `Camera::project()`

Replace the inline `depth_scale` and `lerp(horizon_y, pixel_h, depth_scale)` with `camera.project()`. Verify visual parity by comparing screenshots at idle, mid-speed, and max-speed.

### 6. Migrate sprites.rs

Replace the local projection in `sprites::project()` with `camera.project()` + `camera.project_x()`. Remove `ROAD_MIN_HALF` — road width at horizon emerges naturally from `max_half * depth_scale` at far Z.

### 7. Migrate sky.rs, terrain.rs, effects.rs

Update vanishing point references to use `camera.vanishing_point(screen_w, screen_h)` so speed lines, stars, and terrain all radiate from the same point.
