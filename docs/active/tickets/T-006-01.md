## Context

Currently each git commit spawns a `CommitBillboard` on the roadside verge. This ticket replaces them with small colored racecars placed ON the road surface that the player overtakes.

## Implementation

### 1. Add `CommitCar` variant (`src/world/objects.rs`)

```rust
CommitCar {
    message: String,
    author: String,
    author_color: Rgba<u8>,
},
```

Change `ingest_poll_to_queue` to emit `CommitCar` instead of `CommitBillboard`.

### 2. Add on-road lane positions (`src/world/objects.rs`)

Add `RoadLeft` and `RoadRight` to the `Lane` enum. These use `0.35 * road_half` offset from center (within the road), vs existing `Left`/`Right` at `1.15 * road_half` (on the verge).

### 3. Update spawn logic (`src/world/mod.rs`)

Assign `CommitCar` objects to `RoadLeft`/`RoadRight` lanes (alternating). Other objects keep their existing roadside placement.

### 4. Update projection (`src/renderer/sprites.rs`)

Add `RoadLeft`/`RoadRight` cases to the lane_x calculation in the projection logic. Curvature following is automatic since `cx` already includes `curve_offset * depth²`.

### 5. Add `draw_commit_car()` (`src/renderer/sprites.rs`)

Draw a simplified wedge/racecar shape (~2/3 player car size) colored by `author_color`:
- `car_w < 4px` → colored dot
- `car_w < 8px` → colored rectangle
- Otherwise → full wedge with nose, body dark/light sides

### 6. Message label

Show commit message text above the car when `scale >= 0.5` (close enough to read).

### 7. Z-ordering

No changes needed — existing back-to-front sort handles it. Player car draws after sprites so it's always in front.

## Acceptance Criteria

- [ ] Each commit spawns a colored car on the road surface
- [ ] Cars are colored by author (using `author_color` from RepoSeed)
- [ ] Cars appear in road lanes and follow road curvature
- [ ] Cars grow in perspective as they approach, then pass by
- [ ] Commit message label appears when car is close enough
- [ ] Other roadside objects (AdditionTower, DeletionShard, etc.) remain on the verge
- [ ] Player car always renders in front of commit cars
- [ ] Existing tests updated to reflect CommitCar replacing CommitBillboard
