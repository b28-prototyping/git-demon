## Context

`WorldState` is the simulation core. It tracks camera position, speed, velocity tier, road curvature, and the lifecycle of roadside objects. Each frame calls `update(dt)` to advance the simulation. `ingest_poll()` converts git data into roadside objects.

## Acceptance Criteria

- [ ] `WorldState::new()` initializes from `RepoSeed` with sensible defaults
- [ ] `update(dt)` advances: `z_offset`, `camera_z`, speed lerp toward target, curve offset lerp, tier computation
- [ ] Speed lerps at `4.0 * dt` toward `speed_target = 0.4 + (cpm * 2.8).min(11.6)`
- [ ] `VelocityTier` correctly transitions at thresholds: 0/0.5/1.5/4.0 c/min
- [ ] `ingest_poll()` converts commits to `CommitBillboard`, large additions (>50 lines) to `AdditionTower`, large deletions to `DeletionShard`
- [ ] Objects spawn at `camera_z + SPAWN_DISTANCE` with alternating Left/Right lanes
- [ ] `TierGate` spawns immediately at `camera_z + NEAR_SPAWN` on tier change
- [ ] Road curvature target shifts randomly on activity bursts (>1 c/min)
- [ ] Objects behind camera by `DESPAWN_BEHIND` are removed
- [ ] `draw_distance()` increases 20% at VelocityDemon
