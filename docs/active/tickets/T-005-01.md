## Context

In `src/renderer/road.rs`, `draw_grid` places horizontal neon grid lines at fixed world-Z positions (`z_world = i * grid_spacing`). These positions never change relative to the camera, so as `camera_z` advances, the lines appear to approach but never exit the screen — a Zeno's paradox accumulation effect.

Meanwhile, `draw_road` correctly creates the forward-motion illusion for road stripes via `z_offset / depth` on line 64.

## Bug Analysis

```rust
// Current (broken): grid lines at fixed world positions
let z_world = i as f32 * grid_spacing;  // never moves

// The depth_scale maps this to screen Y:
let depth_scale = (1.0 - z_world / world.draw_distance()).clamp(0.0, 1.0);
let y = lerp(horizon_y as f32, h as f32, depth_scale) as u32;
```

Grid lines are painted at the same screen positions every frame regardless of camera movement.

## Fix

Make grid positions camera-relative by offsetting with `camera_z`:

```rust
let camera_offset = world.camera_z % grid_spacing;
let z_world = i as f32 * grid_spacing - camera_offset;
if z_world <= 0.0 { continue; }
```

This causes lines to scroll toward the camera and wrap around at the grid spacing interval, matching the forward-motion illusion from the road stripes.

## Acceptance Criteria

- [ ] Grid lines scroll toward the camera as `camera_z` advances
- [ ] Lines wrap around smoothly (no pop-in or discontinuity)
- [ ] Lines do not accumulate — the number of visible grid lines stays roughly constant
- [ ] Grid lines still follow curve offset correctly
- [ ] Existing road.rs tests pass
