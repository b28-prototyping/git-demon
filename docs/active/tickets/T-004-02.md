## Context

The rasterizer must hit <4ms per frame at 1920×960. Criterion benchmarks validate this target and catch regressions. The benchmark should exercise the full render pipeline without terminal I/O.

## Acceptance Criteria

- [ ] Criterion benchmark in `benches/render.rs` exercises `FrameRenderer::render()` at 1920×960
- [ ] Benchmark creates a realistic `WorldState` with active objects and non-zero speed
- [ ] Reports per-frame render time in ms
- [ ] Separate benchmarks for individual passes (road, sky, terrain, effects) if total exceeds target
- [ ] No terminal I/O in benchmarks (pure rasterizer measurement)
- [ ] `cargo bench` runs without errors
- [ ] Document results in work directory
