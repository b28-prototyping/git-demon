## Context

High-speed games sell velocity through camera behavior, not just visual effects. The current renderer has a static camera with minor horizon adjustment. This ticket adds physically-motivated camera dynamics that react to speed, curvature, and road slope, making the experience feel embodied.

## Implementation

### 1. FOV widening with speed

As speed increases, widen the effective FOV to compress the periphery and stretch the center — the classic "tunnel vision" of speed.

```rust
camera.fov_scale = 1.0 + speed_normalized * 0.25; // up to 25% wider at max speed
```

This affects `Camera::project()` by scaling `depth_scale` — wider FOV means far objects appear smaller and the road narrows faster.

### 2. Pitch response to slope

When driving uphill, the camera pitches up slightly (seeing more sky); downhill, it pitches down (seeing more road). This should lag behind the actual road slope for a springy feel:

```rust
let target_pitch = -current_segment.slope * 0.04; // radians
camera.pitch += (target_pitch - camera.pitch) * 3.0 * dt; // spring constant
```

The pitch shifts the horizon line and all parallax layers (T-007-03).

### 3. Lateral lag from curvature

When the road curves, the camera should lag slightly behind, creating a lateral offset that shows more of the inside of the curve:

```rust
let target_yaw = -world.curve_offset * 0.002;
camera.yaw_offset += (target_yaw - camera.yaw_offset) * 2.0 * dt;
```

This shifts the vanishing point away from the curve direction, giving the sense of centrifugal force.

### 4. Camera shake at extreme speed

At VelocityDemon tier, add subtle high-frequency shake:

```rust
if tier == VelocityDemon {
    let shake_x = (time * 47.0).sin() * 1.5;
    let shake_y = (time * 31.0).sin() * 0.8;
    camera.yaw_offset += shake_x;
    camera.pitch += shake_y * 0.001;
}
```

Frequencies chosen to avoid resonance with existing sine-wave animations.

### 5. Speed burst zoom

On git activity bursts (commit spike), briefly zoom the camera in then spring back:

```rust
// On burst detection:
camera.fov_scale -= 0.1; // momentary zoom-in
// fov_scale will spring back to speed-based target naturally
```

This creates a "lurch forward" sensation on bursts of git activity.

### 6. Update Camera::update()

Consolidate all camera dynamics into a single `Camera::update(dt, speed, curve_offset, slope, burst)` method that applies all the above with proper spring damping. All parameters lerp toward targets — no discontinuous jumps.

## Acceptance Criteria

- [ ] FOV visibly widens at high speed (road narrows faster toward horizon)
- [ ] Camera pitches with road slope (more sky visible on uphill)
- [ ] Camera lags laterally on curves (vanishing point shifts opposite to curve)
- [ ] Subtle shake at VelocityDemon tier
- [ ] Git activity bursts cause momentary zoom lurch
- [ ] All camera movements are spring-damped (no snapping)
- [ ] Camera dynamics are tunable via constants (easy to adjust feel)
- [ ] No motion sickness: maximum pitch ±2°, maximum lateral offset ±15px at 200w
